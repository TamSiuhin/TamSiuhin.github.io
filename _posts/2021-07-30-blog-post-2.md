# Reinforcement Learning

course from YouTube [Shusen Wang 深度強化學習: Deep Reinforcement Learning](https://www.youtube.com/watch?v=vmkRMvhCW5c&list=PLvOO0btloRnsiqM72G4Uid0UWljikENlU) 

## Basic concepts

### Play Games

1. Observe a frame(State $s_1$)
2. Make action $a_1$ (left, right, up, down)
3. Observe a new frame ($s_2$) and reward $r_1$ 
4. Make action $a_2$
5. ......

### Return and Reward

Definition: cumulative future reward
$$
U_t = R_t + \gamma R_{t+1}+\gamma^2 R_{t+2}+\gamma ^3 R_{t+3}+ ...
$$

* Randomness(action, new state)

### Action-Value Function

Evaluate the action $a_t$ at state $s_t$

Definition: Action-Value function for policy $\pi$
$$
Q_\pi(s_t,a_t) = E[U_t|S_t=s_t, A_t = a_t]
$$
$U_t$ depends on action $A_t, A_{t+1}, A_{t+2},...$ and states $S_t, S_{t+1}, S_{t+2},...$

* action randomness

$$
P[A=a|S=s]=\pi (a|s)
$$

* states randomness

$$
P[S'=s'|S=s,A=a]=p(s'|s,a)
$$

**Optimal action-value function**  

$$
Q^*(s_t,a_t)=\max_\pi Q_\pi(s_t,a_t)
$$

### State-Value Function

$$
V_\pi(s_t) = E_A[Q_\pi(s_t,A)]
$$

score the current state

When actions are discrete  

$$
V_\pi(s_t) = \sum_a \pi (a|s_t)\cdot Q_\pi(s_t,a)
$$

when continuous  

$$
V_\pi(s_t)=\int\pi(s|s_t)\cdot Q_\pi(s_t,a)da
$$


* policy based learning : learn policy function  $\pi(a\vert s)$

* value based learning :  learn action-value function $Q^*(s_t,a)$ 

  $a_t = argmax_a Q^*(s_t,a)$ 

## Value-Based learning 

### Deep Q Network  


$$Q^{*}$$ needs to score every action, and we use neural network to approximate $$Q^{*}$$

### Temporal Difference Learning

$$
Q(s_t,a_t;w)\approx r_t+\gamma \cdot Q(s_{t+1},a_{t+1};w)
$$

$$
U_t = R_t+\gamma\cdot U_{t=1}
$$

Thus,  

$$
Q(s_t,a_t;w)\approx E[R_t+\gamma\cdot Q(S_{t+1},A_{t+1};w)]
$$

$$
E[U_t]\approx E[R_t+\gamma\cdot E[U_{t+1}]]
$$

$$
Q(s_t,a_t;w)\approx r_t+\gamma\cdot Q(s_{t+1},a_{t+1};w)
$$

* Prediction

$$
Q(s_t,a_t;w_t)
$$

* TD target

$$
\begin{align}
y_t&=r_t+\gamma \cdot Q(s_{t+1},a_{t+1};w_t)\\
&=r_t+\gamma\cdot \max_a Q(s_{t+1},a;w_t)
\end{align}
$$

* Loss 

$$
L_t = \frac{1}{2}[Q(s_t,a_t;w)-y_t]^2
$$

* Gradient descent

$$
w_{t+1}=w_t-\alpha \frac{\partial L_t}{\partial w}|_{w=w_t}
$$



## Policy-Based learning 

policy function $$\pi(a\vert s)$$, output is the probability of each action. And then randomly select the action according to the probability.

### Policy Network

approximate $$\pi (a \vert s)$$ 

* use softmax() to output the probability

### State-value function

$$
V_\pi(s_t)=E_A[Q_\pi(s_t,A)]=\sum_a \pi(a|s_t)\cdot Q_\pi(s_t,a)
$$

 use neural network to approximate the policy function  

$$
V(s_t,\theta)=\sum_a \pi(a|s_t;\theta)\cdot Q_\pi(s_t,a)
$$

where $\theta$ is the parameter of policy network  


We need to learn $\theta$ that maximize $J_\theta$  

$$
J(\theta)=E_S[V(S;\theta)]
$$


### Policy gradient ascent

* observe state $s$
* update the police network by

$$
\theta \leftarrow \theta+\beta\cdot\frac{\partial V(s;\theta)}{\partial\theta}
$$

**derive gradient**  (over-simplified)
$$
\begin{align}
\frac{\partial V(s;\theta)}{\partial \theta}&=\frac{\partial \sum_a \pi(a|s;\theta)\cdot Q_\pi(s,a)}{\partial \theta}\\
&=\sum_a \frac{\partial \pi(a|s;\theta)\cdot Q_\pi(s,a)}{\partial \theta}\\
&=\sum_a\frac{\partial\pi(a|s;\theta)}{\partial\theta}\cdot Q_\pi(s,a)\\
&=\sum_a \pi(a|s;\theta)\cdot\frac{\partial log \pi(a|s;\theta)}{\partial\theta}\cdot Q_\pi(s,a)\\
&=E[\frac{\partial log\pi(A|s;\theta)}{\partial\theta}\cdot Q_\pi(s,A)]
\end{align}
$$
if actions are **discrete**  


$$
\frac{\partial V(s;\theta)}{\partial\theta}=\sum_a\frac{\partial\pi(a|s;\theta)}{\partial\theta}\cdot Q_\pi(s,a)
$$


 if actions are **continuous**  


$$
\frac{\partial V(s;\theta)}{\partial\theta}=E_{A\sim\pi(\cdot|s;\theta)}[\frac{\partial log\pi(A|s,\theta)}{\partial\theta}\cdot Q_\pi(s,A)]
$$


**Monte Carlo method**

1. Randomly sample an action $\hat a$ according to the PDF $$\pi(\cdot\vert s;\theta)$$
2. Calculate $$ g(\hat a,\theta)=\frac{log\pi(\hat a\vert s;\theta)}{\partial \theta}\cdot Q_\pi(s,\hat a) $$
3. Use $g(\hat a,\theta)$ as an approximation to the policy gradient  $\frac{\partial V(s;\theta)}{\partial\theta}$

**How to compute $q_t\approx Q_\pi(s_a,a_t)$**

* Reinforce

  * play the game to the end and generate the trajectory

  * compute the discounted return $u_t=\sum_{k=t}^{T}\gamma^{k-1}r_k$

  * since $Q_t(s_t,a_t)=E[U_t]$, we can use $u_t$ to approximate $Q_\pi(s_t,a_t)$

* use neural to approximate $Q_\pi$

  * leads to actor-critic method

##  Actor-Critic Method

combine value-based and policy-based methods

### policy network(actor)

* user network $\pi(a\vert s;\theta)$ to approximate $\pi(a\vert s)$
* athlete 

### value network (critic)

* use network $q(s,a;w)$ to approximate $Q_\pi(s,a)$
* referee (supervision)

### train the network

$$
V(s;\theta,w)=\sum_a \pi(a \vert s;\theta)\cdot q(s,a;w)
$$

* update policy network $\pi(a\vert s;\theta)$ to increase the state-value $V(s;\theta,w)$
* update value network $q(s,a;w)$ to better estimate the return 

### Update value network $q$ using TD

* Compute $q(s_t,a_t;w_t)$ and $q(s_{t+1},a_{t+1};w_t)$
* TD target:

$$
y_t = r_t+\gamma\cdot q(s_{t+1},a_{t+1};w_t)
$$

* Loss

$$
L(w) = \frac{1}{2}[q(s_t,a_t;w)-y_t]^2
$$

* Gradient descent

$$
w_{t+1}=w_t-\alpha\cdot\frac{\partial L(w)}{\partial w}|_{w=w_t}
$$

### Update policy network $\pi$ using policy network

**Policy gradient**

* Let $$g(a,\theta)=\frac{\partial log\pi(a\vert s,\theta)}{\partial \theta}\cdot q(s_t,a;w)$$

* $$\frac{\partial V(s;\theta,w_t)}{\partial \theta}=E_A[g(A,\theta)]$$ 

**Algorithm**

* Randomly sampling: $$a\sim\pi(\cdot\vert s_t;\theta)$$
* Stochastic gradient descent $$\theta_{t+1}=\theta_{t}+\beta\cdot g(a,\theta)$$

## Summary of Algorithm

1. Observe state $s_t$ and randomly sample  $$a_t\sim\pi(\cdot\vert s_t;\theta_t)$$
2. Perform $a_t$ ; then environment gives new states $s_{t+1}$ and reward $r_t$
3. Randomly sample  $\tilde a_{t+1}\sim\pi(\cdot \vert s_{t+1};\theta_{t})$ (do not perform $\tilde a_{t+1}$)
4. Evaluate value network  $q_t=q(s_t,a_t;w_t)$ and $q_{t+1}=q(s_{t+1},\tilde a_{t+1};w_t)$
5. Compute TD error:   $\delta_t = q_t-(r_t+\gamma\cdot q_{t+1})$
6. Differentiate value network:  $$d_{w,t}=\frac{\partial q(s_t,a_t;w)}{\partial w}\vert_{w=w_t}$$
7. Update value network:   $$w_{t+1}=w_t-\alpha\cdot\delta_t\cdot d_{w,t}$$
8. Differnetiate policy network:  $$d_{\theta,t}=\frac{\partial log \pi(a_t\vert s_t,\theta)}{\partial \theta}\vert_{\theta=\theta_t}$$
9. Update ploicy network:  $$\theta_{t+1}=\theta_{t}+\beta\cdot q_t\cdot d_{\theta,t}$$ or $$\theta_{t+1}=\theta_{t}+\beta\cdot \delta_t\cdot d_{\theta,t}$$



## AlphaGo Analysis

**Behavior Cloning + Reinforcement Learning**

### Behavior Cloning (Imitation Learning)

**Similar to classification, regard the ecpert's behavior as labels**

* Observe this state $s_t$
* Policy network makes a prediction:

$$
p_t = [\pi (1\vert s_t,\theta),...\pi(361\vert s_t,\theta)]\in (0,1)^{361}
$$

* The ecpert's action is $$a^* = 281$$
* Loss = CrossEntropy( $y_t$, $p_t$)
* Use gradient descent to update policy network

if current state $s_t$ as not appeared in training data, the action can be **bad**. We need to use reinforcement learning next !

### Reinforcement Learning of policy network

Player(agent)

Opponent(Environment)

* Two policy networks play a game to the end (Player vs. Opponent)

* Get a trajectory: $$s_1, a_1, s_2,  a_2, ......., s_T, a_T$$

* After the game ends, update the player's policy network

  * The player's return: $$u_1 = u_2 = ...... = u_T$$ (either -1 or 1)
  * Sum of approximate policy gradients: 

  $$
  g_\theta=\sum_{t=1}^{T}\frac{\partial log \pi(a_t\vert s_t,\theta)}{\partial\theta}\cdot u_t
  $$

  * Update the policy network:

  $$
  \theta = \theta+\beta\cdot g_\theta
  $$

The learned policy network network is strong,  but not enough. Because a small mistake may change the game result. 

leads to Monte Carlo Tree Search

### State-Value Function

Afer finishing the policy network, train the value network

* Play a game to the end
  * if win, let $$u_1 = u_2 = u_3 =......=u_T = +1$$
  * if lose, let $$u_1 = u_2 = u_3 =......=u_T = -1$$

* Loss

$$
L = \sum_{t=1}^{T}\frac{1}{2}[v(s_t;w)-u_t]^2
$$

* Update

$$
w = w-\alpha\cdot\frac{\partial L}{\partial w}
$$

after training value network, the training process of AlphaGo is finished. Then we need to clarify how AlphaGo make deicision. 

### Monte Carlo Tree Search

 No need to train

* players must look ahead two or more steps

**Main Idea**

* Randomly select an action $a$ (use policy network)
* Look ahead and see whether $a$ leads to win or lose (value function included)
* Choose the action $a$ that has the highest score

**Four Steps:**

1. **Selection:** The player makes an action $a$
2. **Expansion:** The opponent makes an action; teh states updates(imaginary action, made by policy network)
3. **Evaluation:** Evaluate the state-value and get score $v$. Play the game to hte end to receive reward $r$. Assign score $\frac{v+r}{2}$ to action $a$

4. **Backup:** Use the score $\frac{v+r}{2}$ to update action-values

Next, we will explain every steps in detail

#### Step 1: Selection

for all vali actions $a$, calculate the scores
$$
score(a) = Q(a)+\eta\cdot\frac{\pi(a\vert s_t;\theta)}{1+N(a)}
$$

* Given $s_t$, $N(a)$ is how many times we have selected $a$ so far

* $Q(a)$ is Action-value computed by MCTS

#### Setp 2: Expansion

* Given $a_t$, the opponent's action $a_t'$ will lead to the new state $s_{t+1}$
* The opponent's action is randomly sampled from

$$
a_t'\sim \pi(\cdot\vert s_t';\theta)
$$

​	$s_t'$ is the state observed by the opponent

#### Step 3: Evaluation

Run a rollout to the end of the game

* Player's action

$$
a_k\sim \pi(\cdot\vert s_k;\theta)
$$

* Oppoenent's action

$$
a_k'\sim\pi(\cdot\vert s_k';\theta)
$$

* Receive reward $r_T$ at the end

  * if win $r_T = +1$
  * if lose $r_T = -1$ 

* Evaluate the state $s_{t+1}$

  * $$v(s_{t+1};w)$$ : output of the value network

  $$
  V(s_{t+1})=\frac{1}{2}v(s_{t+1};w)+\frac{1}{2}r_T
  $$

#### Step 4: Backup

* MCTS repeats such a simulation many times
* Each child of $a_t$ has multiple reord $V(s_{t+1})$
* Update action -value

$$
Q(a_t)=\text{mean(the recorded}\;V's)
$$

* The $Q$ values will be used in Step 1(selection)

#### Decision Making

* $N(a)$: How many times $a$ has been selected so far
* After MCTS, the players maeks actual decision

$$
a_t = \arg \max_a N(a)
$$

$N(a)$ is how many times action $a$ is selected

* To perform the next action, AlphaGo do MCTS all over again (Initialize $Q(a)$ and $N(a)$ to zeros)
